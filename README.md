# AustralScience — User Feedback Intelligence Platform

This repository folder is a **portfolio showcase** for the **AustralScience JustWatch demo**.

- **Live demo)**: `https://justwatch.australscience.com`
- **Important**: The demo website displays **fake / synthetic data generated by AI** for showcasing purposes.  
  No client production data is used here.

### Problem

- Company is scaling → a UI update was expected to improve engagement.
- Engagement **did not increase** as expected (it declined).
- Internal engagement data showed **where** users dropped off, but not **why**.
- We needed to connect **user behavior** with **user perception**.

### My role / How I solved it

- Combined internal engagement data with external social media feedback.
- Cleaned, validated, and integrated datasets (SQL + Python).
- Performed sentiment + topic analysis to track perception over time.
- Designed more meaningful metrics than basic sentiment:
  - aggregated posts + comments at **thread level**
  - applied **weighted scoring** (higher-engagement content = higher impact)
- Used **LLM + prompt engineering** for sentiment and topic classification.
- Built a web-based interactive dashboard and created usage documentation with Product & Growth teams.

### Challenge

- No standard sentiment formula aligned with business goals.
- Defined metrics from scratch:
  - how to combine posts and comments
  - how to structure thread-level analysis
  - how to weight popularity and engagement
- Iterated through approaches and validated with stakeholders (balance technical rigor vs business relevance).

### Result / Insight

- Negative sentiment increased within **24–48 hours** after release.
- Key issues: **navigation**, **content discoverability**, and UX friction.
- Sentiment trends aligned with engagement drop → explained the **“why”** behind the behavioral change.

### Business impact

- Product: identified specific UX issues to prioritize improvements.
- Growth: gained an early signal of user dissatisfaction.
- Established a reusable framework for post-release monitoring.

### Improvements (if more time)

- Improve prompt design and compare multiple LLM approaches.
- Strengthen validation for weighted sentiment metrics.
- Ensure robustness across more releases and data patterns.

## My scope (Data Analyst)

The focus of this showcase is my analytics contribution:

- Analysed **50K+** user feedback records by combining structured engagement data and unstructured user comments to identify behavioural trends and shifts in user perception, directly informing product prioritization decisions for the Growth team.
- Detected a spike in negative sentiment within **24–48 hours post-release**, uncovering key UX issues related to navigation and content discoverability, reducing time to identify UX issues from **~5-7 days to ~1-2 days (~2-3x faster)** based on internal team feedback.
- Cleaned and validated datasets by systematically handling noise, duplicate records, missing values, and inconsistent data formats to ensure reporting accuracy and reliability for downstream analysis.
- Designed and maintained SQL queries in **PostgreSQL** to enable reliable reporting for weekly business reviews.
- Applied **LLM-assisted** sentiment and keyword analysis (LangChain, Groq API) to monitor engagement trends and surface recurring UX pain points, significantly reducing manual tagging effort.
- Built and enhanced shared analytical dashboards with scheduled data refresh, enabling teams to access up-to-date engagement & sentiment KPIs and reducing ad-hoc reporting requests.
- Presented weekly insights to the Growth Team Lead and non-technical stakeholders, translating analysis into clear narratives that influenced product roadmap decisions.
